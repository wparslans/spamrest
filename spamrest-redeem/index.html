⚠️

Webhook caveat — read this. AppSumo validates the webhook with a POST, but GitHub Pages only answers GET (a POST returns 405). So the webhook URL cannot pass validation on Pages alone. Point the Webhook URL at a tiny serverless function instead (Cloudflare Workers / Netlify / Vercel) — copy-paste handler below — or keep your existing WordPress REST route. The OAuth Redirect URL above works on Pages with no backend.

Cloudflare Worker — drop-in webhook receiver (returns the exact JSON AppSumo expects)
javascript
export default {
  async fetch(request) {
    if (request.method === 'GET') {
      return Response.json({ status: 'ok', success: true });
    }
    if (request.method === 'POST') {
      const body = await request.json().catch(() => ({}));
      const event = body.event || 'purchase';
      // TODO: persist body.license_key, handle event, verify HMAC signature.
      return Response.json({ event, success: true });
    }
    return new Response('Method Not Allowed', { status: 405 });
  }
};
